[今回のC言語クライアントの概要]

・利用するVBコンポーネントについて
　今回は、Visual Basic 6.0 の ActiveX DLL として作成した COMコンポーネントを、 C言語クライアントからアクセスしてみます。Visual Basic で作成した ActiveX DLLの概要は以下のとおりです。メソッドは、二つの引数を受け取り、加算した結果を戻り値として返します。

　プロジェクトの種類: ActiveX DLL
　プロジェクト名　　: Project1 (ダウンロード:project1.lzh 同梱のReadMe.txtをお読みください)
　クラスモジュール名: Class1
　メソッド: 
    'メソッドCalc
    Public Function Calc(ByVal x As Long, ByVal y As Long) As Long
        Calc = x + y
    End Function
　これを COMの観点から見ると、以下の特徴を持つコンポーネントであるといえます。

　ProgID: Project1.Class1
　クラス名: Class1
　クラスID: (Visual Basicでは見えない)
　インターフェイス: _Class(dualインターフェイス)
　　　　　　　　　　つまり、IDispatchから継承して、Calcメソッドを追加されたインターフェイス

　この COMコンポーネントに対して、もし VBScriptからアクセスする場合、以下のような記述になります。 
    'インスタンスの作成と呼び出し
    Dim obj, x, y, z
    Set obj = CreateObject("Project1.Class1")  'インスタンスの作成、Project1.Class1 は ProgID
    x = 10
    y = 20
    z = obj.Calc( x, y ) 'メソッドの呼び出し
    Set obj = Nothing    'インスタンスの破棄
　この VBScriptと同様のアクセスを C言語から行ってみます。
　C言語からアクセスするには、COMコンポーネントを識別するための ProgIDまたはCLSID(クラスID)によるコンポーネントを特定する作業と、利用するインターフェイスを特定する作業が必要です。
　今回のコードでは、できるだけ理解しやすくするために、可能な限りマクロの使用は抑えました。また、省けるステップはできるだけ省きました。
